Seam Wallet API

This API allows you to get and alter the balance of users' wallets managed on the IFRAME Service side.

For these purposes, your service must send requests to the IFRAME Service along with a token unique for the user and a wallet.

Getting the balance. Integration flow

You need to:

  1. (optionally) Generate unique token1 and sign (see Request signature).

  2. Call /GetBalance method.

  3. IFRAME Service will send a /GetUserData request to your host address.

    Your service should validate the token and signature and return data, including the existing user's userId.

  4. IFRAME Service will send a response that contains the balance of the wallet tied to the user.

Getting the balance. Integration flow

You need to:

  1. (optionally) Generate unique token1 and sign (see Request signature).

  2. Call /TransferFunds method with operation parameters including transferId.

    info

    Requests with the same transferId and other parameters are idempotent, meaning that if the first request was successful, the service will return a response with a status of 409 for all subsequent requests.

    If in repeat requests the transferId matches, but other parameters differ, they will be rejected with 400 status code. One transferId cannot be used for different userId.

  3. IFRAME Service will send a /GetUserData request to your host address.

    Your service should validate the token and signature and return data, including the existing user's userId.

  4. In the case of success, IFRAME Service will respond with 200 status code.
